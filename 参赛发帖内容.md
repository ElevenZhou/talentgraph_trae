# 【Code with SOLO】用 SOLO 5 天从零搭建一个 AI 原生人才图谱平台，让简历变成 AI 可读的能力宇宙

## 摘要

用 TRAE SOLO 从零搭建了 **TalentGraph（人才引力场）**——一个 AI 原生人才能力图谱平台。上传传统简历，AI 自动解析为结构化能力图谱（D3.js 三层径向可视化），再与 50+ 真实岗位智能匹配。作为产品/运营角色，用 SOLO 完成了包含用户系统、AI 解析、图谱可视化、智能匹配、运营后台的完整全栈项目。

## 背景

我是一名产品/运营，日常工作中经常需要做竞品分析和人才评估。传统简历是给人类 HR 快速浏览的，信息密度低、格式不统一，很难被系统化处理。

我一直有个想法：**如果简历能被 AI 理解，变成结构化的能力数据，那人才和项目的匹配就可以自动化了。**

但我是产品/运营背景，虽然有一些技术理解，要从零写一个完整的 Web 应用还是很有挑战的。之前用其他 AI 编程工具试过，但要么生成的代码跑不起来，要么做到一半就卡住了。

这次用 TRAE SOLO，我决定认真试一把，看看能不能真正把想法变成可运行的产品。

## 实践过程

### 第一步：拆解任务

我没有一上来就让 SOLO 写代码，而是先把自己的想法梳理清楚：

1. **核心价值**：把简历变成 AI 可读的结构化数据
2. **关键功能**：AI 解析 → 能力图谱可视化 → 岗位匹配
3. **技术选型**：让 SOLO 推荐适合的方案
4. **产品闭环**：用户系统 + 核心功能 + 管理后台

然后我按模块逐步推进，每个模块都是一个独立的对话。

### 第二步：搭建基础框架

我告诉 SOLO：

> "我要做一个 Next.js 项目，需要用户注册登录、AI 简历解析、图谱可视化、岗位匹配功能。请帮我搭建项目框架。"

SOLO 帮我选择了：
- **Next.js 14 (App Router)** + TypeScript — 现代前端框架
- **Tailwind CSS** — 样式方案
- **SQLite (better-sqlite3)** — 轻量本地数据库，数据不出境
- **NextAuth.js** — 认证系统
- **D3.js** — 图谱可视化
- **DeepSeek + OpenAI** — 双 AI 服务商

### 第三步：实现 AI 简历解析（核心功能）

这是整个项目最核心的部分。我给 SOLO 的关键 Prompt 是：

> "用户上传简历文本，调用 DeepSeek API 解析，输出结构化 JSON，包含身份信息、能力列表（含等级和评分）、项目证据、能力边界评估、匹配建议。用 SSE 流式返回进度。"

SOLO 帮我实现了：
- PDF 文件上传解析（pdfjs-dist）
- 文本粘贴解析
- AI 结构化输出（严格 JSON Schema）
- SSE 流式进度展示（6 个步骤动画）
- 双 AI 服务商切换（DeepSeek / OpenAI）

### 第四步：D3.js 三层径向图谱可视化

这一步是最有挑战的。我想要的效果是：

> "中心是人物身份，第一圈是核心能力，第二圈是项目证据，用不同颜色区分能力类别，节点大小反映能力强度，连线粗细反映关联度。"

SOLO 用 D3.js 实现了三层径向布局：
- **核心层**：身份信息（姓名、角色、城市）
- **能力层**：按类别（产品/技术/分析/协作/管理/工具）分布，颜色区分
- **证据层**：项目经历、工作经历，连线到对应能力

还支持节点点击查看详情、hover 高亮关联节点。

### 第五步：智能匹配系统

我准备了 50 个类真实岗位数据（覆盖 AI、互联网、金融、医疗等 12 个行业），SOLO 帮我实现了：

- 多维度匹配算法（技能匹配度、经验匹配度、行业匹配度）
- 匹配结果按分数排序
- 详细匹配分析（优势、缺口、风险、建议）
- 支持 3 个 Demo 体验（产品经理、全栈工程师、应届生）

### 第六步：运营后台

最后补上了管理后台：
- 数据仪表盘（用户数、简历数、匹配次数）
- 用户管理（搜索、筛选）
- 简历管理（查看、删除）
- AI 配置（服务商切换、API Key 管理、连接测试）

### 踩过的坑

1. **D3.js 整包引入导致编译巨慢**：第一次用 `import * as d3 from 'd3'`，开发模式每个页面首次访问要等 10+ 秒。后来了解到这是开发模式的正常行为，生产环境不受影响。

2. **AI 返回格式不稳定**：DeepSeek 有时会在 JSON 外面包 markdown 代码块，导致解析失败。SOLO 帮我在 Prompt 里加了严格约束，并在代码里做了容错处理。

3. **PDF.js Worker 路径问题**：Next.js 打包后 PDF.js 的 worker 文件路径会变，SOLO 帮我用动态路径解决了。

4. **Session 在页面间丢失**：NextAuth 的 JWT session 在某些页面跳转时丢失，排查发现是 SessionProvider 没正确包裹布局组件。

## 效果与总结

### 提效数据

| 对比项 | 传统方式 | 用 SOLO |
|--------|---------|---------|
| 开发周期 | 2-4 周（需要找开发） | 5 天 |
| 技术门槛 | 需要全栈开发能力 | 产品运营也能做 |
| 代码质量 | - | TypeScript 严格类型，架构清晰 |

### SOLO 在我的流程中做了什么

1. **技术选型**：我不确定用什么技术栈，SOLO 根据需求推荐了最合适的方案
2. **代码生成**：从项目框架到每个功能模块，SOLO 生成了可直接运行的代码
3. **问题排查**：遇到 bug 时，SOLO 能快速定位原因并给出修复方案
4. **架构设计**：AI 服务双服务商、SSE 流式、Agent API 等设计都是和 SOLO 讨论出来的

### 可复用的方法

1. **先拆解再执行**：不要一上来就写代码，先把需求拆成独立模块
2. **一个模块一个对话**：每个功能模块在独立对话中完成，上下文更清晰
3. **准备好测试数据**：提前准备 Demo 数据，方便验证功能
4. **截图记录过程**：每完成一个功能就截图，方便后续写文档
